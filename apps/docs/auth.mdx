title: Authentication
description: Better Auth with bearer tokens and TOTP two-factor.
---

Auth is handled by [Better Auth](https://better-auth.com). `@medai/auth`
configures the server instance with the Prisma adapter and the `twoFactor`,
`bearer`, and `jwt` plugins; the API mounts it under `/api/auth/*`.

## Token strategy

The **bearer** plugin lets API clients send `Authorization: Bearer <token>`
instead of relying on cookies. The **jwt** plugin additionally exposes signed
JWTs (and a JWKS endpoint) for service-to-service verification. Sessions are
stored in Postgres via Prisma.

## Endpoints

Better Auth exposes the full set under `/api/auth/*`, including:

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/api/auth/sign-up/email` | Create an account, returns a token |
| `POST` | `/api/auth/sign-in/email` | Email + password → token |
| `GET` | `/api/auth/get-session` | Resolve the current session |
| `POST` | `/api/auth/sign-out` | Revoke the session |
| `POST` | `/api/auth/two-factor/enable` | Begin TOTP enrollment |
| `POST` | `/api/auth/two-factor/verify-totp` | Verify a TOTP code |

## Example

```bash
curl -X POST localhost:4000/api/auth/sign-up/email \
  -H 'Content-Type: application/json' \
  -d '{"email":"u@example.com","password":"SecurePass123!","name":"User"}'

curl localhost:4000/api/auth/get-session -H 'Authorization: Bearer <token>'
```

## Protecting routes

The API's `requireAuth` middleware validates the session via Better Auth and
populates `req.user` (`id` + `role`). The `role` is a custom additional field
on the user (`patient` by default).

## Client

The web app uses the Better Auth React client (`apps/web/lib/auth-client.ts`):

```ts
import { authClient, signIn, useSession } from '@/lib/auth-client';
```
